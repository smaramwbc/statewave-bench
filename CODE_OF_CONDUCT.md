# Code of Conduct

Statewave adopts the **Contributor Covenant, version 2.1** as its Code of Conduct.

- Full text: https://www.contributor-covenant.org/version/2/1/code_of_conduct/
- FAQ: https://www.contributor-covenant.org/faq
- Translations: https://www.contributor-covenant.org/translations

## Scope

This Code applies to all Statewave repositories under [github.com/smaramwbc](https://github.com/smaramwbc), to issues and pull requests across those repositories, to [GitHub Discussions on the core repo](https://github.com/smaramwbc/statewave/discussions), and to any official Statewave community channel.

It applies equally when contributors represent Statewave in public spaces — for example, posting from a Statewave-affiliated account or speaking at an event on the project's behalf.

## Reporting

Concerns about contributor behavior can be sent privately to:

> **conduct@statewave.ai**

Reports are read by the project maintainers and handled in accordance with the **Enforcement Guidelines** in Contributor Covenant 2.1.

If a concern involves a project maintainer directly, you may instead email [security@statewave.ai](mailto:security@statewave.ai), which is monitored separately.

## Enforcement

Project maintainers follow the Contributor Covenant 2.1 Enforcement Ladder when responding to reports — correction, warning, temporary ban, permanent ban — based on the severity and pattern of the conduct involved.

Maintainers who do not follow or enforce this Code in good faith may face temporary or permanent repercussions as determined by other members of the project's leadership.
