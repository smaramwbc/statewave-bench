# Support

Welcome to `statewave-bench` — the open benchmark for AI memory runtimes (Statewave vs Mem0 vs Zep on LoCoMo). Here's how to get help.

## 📖 Documentation

Start with the docs:

- **[README](README.md)** — what the bench measures, how to run it, headline numbers
- **[Statewave docs](https://github.com/smaramwbc/statewave-docs)** — for the runtime under test
- **[Statewave examples](https://github.com/smaramwbc/statewave-examples)** — runnable agent demos

## 💬 Community

For questions and discussions:

- **[GitHub Discussions on the core repo](https://github.com/smaramwbc/statewave/discussions)** — ask questions about the bench, results interpretation, or new evaluations to add

## 🐛 Bug Reports

`statewave-bench` Issues are **disabled** in favour of a single central tracker. File bench bugs here:

> **[smaramwbc/statewave issues](https://github.com/smaramwbc/statewave/issues/new/choose)**

Use the prefix `[bench]` in the title and include:

1. Reproduction command (the exact `swb run` invocation)
2. Adapter / system the bug affects
3. Dataset commit + relevant `uv.lock` SDK versions
4. Expected vs. actual rows in the JSONL output

## ✨ Feature Requests

New adapter, new metric, new dataset? Discuss before building:

1. Check open [Discussions](https://github.com/smaramwbc/statewave/discussions) — someone may already be working on it
2. Open a new discussion under **Ideas & Feature Requests** describing the use case

## 🔒 Security Issues

**Do NOT open public issues for security vulnerabilities.**

Email [security@statewave.ai](mailto:security@statewave.ai) instead. See [SECURITY.md](SECURITY.md) for the coordinated disclosure process.
